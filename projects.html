DNS Attacks
DNS Pharming
Three-part DNS attack simulation: HOSTS file modification, DNS response spoofing (Wireshark), and DNS server cache poisoning — all verified via dig and Wireshark.
DNS Wireshark dig Cache Poisoning
Click to expand

Part 1 — HOSTS File Modification

  • Edited the HOSTS file to override DNS for www.netsec-week3.com
  • Verified the change using the dig command

Part 2 — DNS Response Spoofing

  • Captured DNS query from the client using Wireshark
  • Crafted and sent a forged DNS response from the attacker VM
  • Observed the spoofed result in Wireshark before stopping the attack

Part 3 — DNS Server Cache Poisoning

  • Injected a forged DNS response into the DNS server's cache
  • Subsequent client queries answered using the poisoned cache entry
  • Verified success using dig and Wireshark

Setup

Security Policy Configuration

  • Configured policy to reject Telnet access to the server between zones
  • Set iptables rules to permit HTTP and HTTPS, dropping all other traffic

Firewall User Authentication

  • Implemented pass-through authentication using HTTP/HTTPS
  • Added custom banners for improved user experience

IDS Configuration

  • Configured UDP flood threshold at 1024 packets
  • Further UDP packets beyond threshold flagged and logged
